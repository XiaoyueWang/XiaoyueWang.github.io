开启闲散的写作时代……

日志
  • Github Pages
  • Hexo
  • Mac

前言

作为博客的第一篇文章，就写一写这篇博客的搭建方法吧，按照本篇记载的方式一步一步的做，即可做出一样精美的本博客了，而且过程还很简单。

这里采用的是 GitHub Pages + GitCafe + Hexo 混合搭建的方式

  • GitHub Pages
  • Hexo

GitHub Pages

看名字就知道是GitHub自家的东西，把静态文件托管到GitHub上，把它当成一个普通的项目维护就可以了。 下面简单的几步就可以实现了：

  1. 首先创建GitHub同名项目，格式为：username.github.io。

    If the first part of the repository doesn’t exactly match your username, it won’t work, so make sure to get it right.

    如果项目名称第一部分和用户名不一致，这个项目是不会起作用的，所以确保正确性。

  2. 接下来就是git基本操作了，clone项目、创建“Hello World”测试文件、提交项目。

    推荐一款好用的git客户端SourceTree，图形界面使用起来方便快捷。当然，如果想保持程序员的低调的性格，请继续使用git命令操作。

    Clone the repository
    进入到存放项目的目录下，执行命令： 

    ~ $ git clone https://github.com/username/username.github.io

    Hello World
    进入项目目录，创建index.html

    ~ $ cd username.github.io
~ $ echo "Hello World" > index.html

    Push it
    添加、提交并且推送修改的内容

    ~ $ git add --all            
~ $ git commit -m "Initial commit"
~ $ git push -u origin mast``

    …and you’re done!
    完成了，是不是很简单？http://username.github.io 去看看效果吧（记得把username修改成你自己的用户名）！
    如果看到了“Hello World”，祝贺你成功了！至此，完成了GitHub Pages的相关配置。

Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。关于Hexo 更多信息，大家可以去官网慢慢查看，这里我只是介绍部署涉及的基本配置。
Hexo 的配置，还是有一些需要注意的关键点，只是看文档，并不一定能成功，请大家注意！

安装

Hexo 的安装相当简单，但是需要优先安装 Node.js 和 Git

安装 Git

如果你是一名iOS Developer，那么你的电脑中一定有Xcode，恭喜，可以忽略Git的安装了，因为在安装Xcode的时候会同时安装Git。
如果没有Git，可以通过以下方式安装：

  • Windows：下载并安装 Git
  • Mac：使用 Homebrew, MacPorts 或下载 安装程序 安装
  • Linux (Ubuntu, Debian)：sudo apt-get install git-core
  • Linux (Fedora, Red Hat, CentOS)：sudo yum install git-core

安装 Node.js

安装 Node.js 的最佳方式是使用 nvm，方法如下：
cURL: 

~ $ curl https://raw.github.com/creationix/nvm/master/install.sh | sh

Wget:

~ $ wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh

安装完成后，重启终端并执行下列命令即可安装 Node.js。

~ $ nvm install 4

或者您也可以下载 安装程序 来安装。

注意：
在使用命令安装时，可能会遇到抱错的情况，主要就是一些依赖的包或者软件不存在或者版本过低，只需要认真阅读抱错信息，安装或升级对应的包和软件即可。

安装 Hexo

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

~ $ npm install -g hexo-cli

如果你的电脑中已经安装好了 Node.js 和 Git，接下来只需要使用下面的命令，就可以完成 Hexo 的安转了：

~ $ npm install -g hexo-cli

建站

安装完成后，需要执行下面的命令进行建站，生成相关的文件：

~ $ hexo init <folder>
~ $ cd <folder>
~ $ npm install

注意：这里的folder，不是Github pages的目录，而是一个专门存放Hexo相关文件的目录，可以起名叫Hexo，也可以叫blog

创建完成后，floder的目录如下： 

.  
├── _config.yml  
├── package.json  
├── scaffolds  
├── source  
|   ├── _drafts  
|   └── _posts  
└── themes
  • _config.yml：网站的 配置 信息，可以在此配置大部分的参数。
  • package.json：应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。
  • scaffolds：模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
  • source：资源文件夹是存放用户资源的地方。除 posts 文件夹之外，开头命名为 (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹(待会会讲到如何生成public文件夹)，而其他文件会被拷贝过去。
  • themes：主题 文件夹。Hexo 会根据主题来生成静态页面。

配置

官网说的很明白了，直接照着做就可以了。需要注意一点，如果部署的时候出现抱错了，可能是配置文件中的空格导致的，只需要根据抱错信息提示的具体行数，仔细检查一下就可以了。
配置完成后，我们可以启动本地服务来看一下效果了，执行命令。

~ $ hexo s --debug

如果没有抱错并且看到下面的提示信息时，表明启动成功了，在浏览器中输入 http://0.0.0.0:4000/ 就可以看到效果了。

INFO  Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

当然，这只是本地服务的效果，最终想要上传到Github pages 上，还需要继续下面的生成文件和部署。

生成文件

每次新建文件或者修改配置都需要生成静态文件，使用如下命令。

~ $ hexo generate

选项                描述
-d, --deploy    文件生成后立即部署网站    
-w, --watch        监视文件变动

执行完成后，会发现在 Hexo 目录下出现了一个public文件夹，这里面放的都是生成的静态文件，也就是将来要发布到Github 上的。

部署

Hexo 支持Git、Heroku、Rsync、OpenShift、FTPSync。每个在 _config.yml 中的配置都不一样，因为我用的是Git，这里是说一下 Git 的配置方式，至于其他的请大家自己查看官网吧
需要 _config.yml 在中配置部署的参数。

deploy:
     type:github    ##指定部署服务器的类型。官网说使用git，这里我用的是github。
      repository:https://github.com/XiaoyueWang/XiaoyueWang.github.io.git  ##Github pages的地址
      branch:master    ##指定项目的branch

完成参数的配置，执行下面的命令进行部署。

~ $ hexo deploy

生成文件和部署一次性完成，下面两条命令作用是一样的。

  ~ $ hexo generate --deploy
~ $ hexo deploy --generate

发布到Github

部署完成后，需要把静态文件传到我们的 Github pages 上，也就是一开始创建的 username.github.io 项目。

注意：只需要上传 public 文件夹即可，千万不要把整个 Hexo 全部上传，那样是不会看到你想要的效果的。

主题

作为个人博客来讲，个性的主题尤为重要，官网主题这部分主要介绍了如何制作自己的主题，这个难度比较大，我们可以选择站在居然的肩膀上，直接使用其他人已经创建好的主题。所有的主题都存放在themes目录中。我用的Yelee，很漂亮，部署简单，具体方法这里就不介绍了。Hexo提供的主题有很多，大家可以根据自己的兴趣自行选择。

写作

博客最主要的，当然还是写东西啦。Hexo 创建新文章非常简单，只需要执行下面一条命令。

~ $ hexo new [layout] <title>

布局（layout）

Hexo 有三种默认布局：post、page 和 draft，它们分别对应不同的路径，如果自定义的其他布局和 post 相同，都将储存到 source/_posts 文件夹。若不指定，默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。

post: source/_posts
page: source
draft: source/_drafts

操作流程

上面说了那么多，好多都是安装部署的介绍，其实真正使用起来还是很简单的，总结下来也就下面的几步而已。

  1. 启动本地服务，查看修改后的效果。

    ~ $ hexo deploy

  2. 清除缓存文件 (db.json) 和已生成的静态文件 (public)。

    ~ $ hexo clean

  3. 生成文件，生成public目录中的静态文件。 

    ~ $ hexo generate

  4. 部署

    ~ $ hexo deploy

  5. public 目录上传到Github pages

    只需要将public目录上传到 Github pages ，访问http://username.github.io，是不是您想要的效果呢 ？

最后
